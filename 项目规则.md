# 项目自动化测试规范

## 1. 技术栈
- 语言: Python 3
- 测试框架: pytest
- HTTP请求库: requests
- 报告生成: pytest-html

## 2. 目录结构
- `tests/`: 存放测试用例文件，以 `test_` 开头。
- `utils/`: 存放公共工具类，如API封装、配置读取等。
- `reports/`: 存放生成的HTML测试报告。
- `run.py`: 测试执行入口脚本。

## 3. 命名规范
- 测试文件: `test_*.py`
- 测试类: `Test*`
- 测试方法: `test_*`

## 4. 报告规范
- 必须生成独立的HTML报告。
- 报告应包含用例描述、运行时间、错误日志等。
